# r8152 DKMS Driver — RTL8156 USB NIC

**Date:** 2026-03-14
**Hosts:** bupu, tika (Proxmox nodes with new USB network adapters)
**Beads:** homelab-dd0

## Problem

New USB NICs (Realtek RTL8156, vendor `0bda`, product `8157`) installed on `bupu` and `tika`
default to the in-kernel `cdc_ncm` driver, which lacks RTL8156 support and does not support
jumbo frames. This was contributing to HTTP 595 / Connection refused errors observed during
Terraform apply against those nodes.

## Solution

Install the out-of-tree Realtek r8152 driver via DKMS from the actively maintained fork:
[wget/realtek-r8152-linux](https://github.com/wget/realtek-r8152-linux)

The Ansible role `roles/r8152`:
- Skips gracefully if no `0bda:8157` device is found via `lsusb`
- Builds and installs the driver via DKMS (survives kernel upgrades)
- Writes `0bda 8157` to `/sys/bus/usb/drivers/r8152/new_id` for immediate runtime binding
- Deploys `/etc/udev/rules.d/50-r8152-rtl8156.rules` for persistent binding across reboots
  - Unbinds cdc_ncm from this specific USB interface (per-device, not a global blacklist)
  - Binds to r8152

## Deploy

```bash
task r8152               # targeted run (proxmox group, r8152 tag)
task r8152 -- --check    # dry-run first
task proxmox             # full proxmox run (includes r8152 role)
```

## Validation

### 1. Confirm driver binding

```bash
# Find the interface name for the USB NIC
ip link show | grep enx

# Verify the kernel driver in use
ethtool -i enx<mac>
# Expected: driver: r8152
```

### 2. Test jumbo frame support (MTU 9000)

```bash
ip link set enx<mac> mtu 9000
ip link show enx<mac>
# Confirm: mtu 9000

# Ping test across the storage VLAN with large frames
ping -M do -s 8972 <storage-vlan-gateway>
```

### 3. Test NFS over storage VLAN with jumbo frames

```bash
# On the Proxmox node, confirm proxmox-nfs storage is accessible
pvesm status | grep proxmox-nfs
# Expected: proxmox-nfs  nfs  active

# Run a quick I/O test to the NFS datastore
dd if=/dev/zero of=/mnt/pve/proxmox-nfs/test.img bs=1M count=512 oflag=direct
# Note throughput; compare to pre-jumbo-frame baseline
rm /mnt/pve/proxmox-nfs/test.img
```

### 4. Verify DKMS survives kernel upgrade

```bash
dkms status
# Expected: r8152/<version>, <kernel>, x86_64: installed
```

## Rollback

```bash
# Remove DKMS module
dkms remove r8152/<version> --all

# Restore default driver binding
rm /etc/udev/rules.d/50-r8152-rtl8156.rules
udevadm control --reload-rules

# Rebind to cdc_ncm
echo "0bda 8157" > /sys/bus/usb/drivers/cdc_ncm/new_id
```
